# Правила для backend-разработчика

## Разделение зон ответственности

- **Ты** работаешь ТОЛЬКО в папке `backend/`
- **Напарник** работает ТОЛЬКО в папке `frontend/`

## СТРОГИЕ ЗАПРЕТЫ

- НИКОГДА не редактируй, не создавай и не удаляй файлы в папке `frontend/`
- НИКОГДА не меняй `frontend/package.json`, `frontend/src/` или любые другие файлы фронтенда
- Если задача требует изменений на фронтенде — сообщи об этом пользователю, но НЕ вноси изменения сам
- Не запускай команды из папки `frontend/`

## Разрешённые зоны

- `backend/` — основная рабочая зона
- `README.md` — можно обновлять раздел про API

## Git-воркфлоу

- Перед началом работы: `git pull origin master`
- Создавай ветку: `git checkout -b feature/название-фичи`
- Коммить только файлы из `backend/`
- Пуш и Pull Request в master
