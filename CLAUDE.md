# Правила проекта Tatto-studya

## Разделение зон ответственности

Этот проект разрабатывается двумя людьми:
- **Frontend-разработчик** (владелец этого окружения) — работает ТОЛЬКО в `frontend/`
- **Backend-разработчик** (напарник) — работает ТОЛЬКО в `backend/`

## СТРОГИЕ ЗАПРЕТЫ

- НИКОГДА не редактируй, не создавай и не удаляй файлы в папке `backend/`
- НИКОГДА не меняй `backend/package.json`, `backend/prisma/schema.prisma` или любые другие файлы бэкенда
- Если задача требует изменений в бэкенде — сообщи об этом пользователю, но НЕ вноси изменения сам
- Не запускай команды `npm install`, `npm run` и т.д. из папки `backend/`

## Разрешённые зоны

- `frontend/` — основная рабочая зона (создание, редактирование, удаление файлов)
- `README.md` — можно обновлять
- `.gitignore` — можно обновлять
- Корневые конфиги (если появятся) — только по согласованию

## Git-воркфлоу

- Перед началом работы: `git pull origin master`
- Создавай ветку: `git checkout -b feature/название-фичи`
- Коммить только файлы из `frontend/` и корневые файлы
- Пуш и Pull Request в master
